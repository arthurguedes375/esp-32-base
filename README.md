# ESP32 Template Project

This repository provides a robust template for developing applications for ESP32 microcontrollers using Rust and the ESP-IDF framework. It's designed to be a ready-to-use starting point, pre-configured with essential components like Wi-Fi connectivity, and a well-structured codebase to ensure a smooth development experience.

## Quick Start: Compile and Run

Follow these steps to get your project compiled and running on an ESP32 device.

1.  **Initial Build:** Run the build command. This first attempt will likely fail but prepares the necessary directories.
    ```bash
    cargo build
    ```

2.  **Copy Partitions File:** Manually copy `partitions.csv` to the build output directory. *Verify the exact path from your build error message.*
    ```bash
    cp partitions.csv /var/robot/autocar/code/target/xtensa-esp32-espidf/debug/build/esp-idf-sys-ee16df87bc2e2aef/out/
    ```

3.  **Second Build:** Run `cargo build` again. It should now succeed.
    ```bash
    cargo build
    ```

4.  **Flash and Monitor:** Flash the binary to your ESP32 and open the serial monitor.
    ```bash
    espflash flash --monitor -S --no-verify
    ```

For more detailed instructions, refer to `docs/001-running.md`.

## Project Specifications

*   **Target Device:** ESP32
*   **Programming Language:** Rust
*   **Framework:** ESP-IDF (via `esp-idf-sys` crate)
*   **Key Features:** Basic Wi-Fi connectivity (template-level, implementation details within `src/logical` or `src/main.rs`), structured for embedded application development.

## Folder Structure

The project follows a logical and conventional structure:

*   `.cargo/`: Rust cargo configuration.
*   `.git/`: Git version control metadata.
*   `.vscode/`: VS Code editor configuration files.
*   `docs/`: Documentation files, including setup and usage guides.
*   `src/`: Source code for the application.
    *   `main.rs`: The entry point of the Rust application.
    *   `logical/`: Contains higher-level application logic and business rules.
        *   `mod.rs`: Module declaration for logical components.
    *   `physical/`: Contains drivers and interfaces for physical hardware components.
        *   `buzzers/`: Buzzer driver module.
        *   `leds/`: LED driver module.
        *   `motors/`: Motor control modules.
            *   `dc.rs`: DC motor specific implementations.
*   `target/`: Compilation output directory (generated by Cargo).
*   `build.rs`: Custom build script for Rust projects.
*   `Cargo.toml`: Rust project manifest and dependency management.
*   `diagram.json`: Potentially a diagram definition file (e.g., for Wokwi or another diagramming tool).
*   `partitions.csv`: ESP-IDF partition table definition.
*   `rust-toolchain.toml`: Specifies the Rust toolchain version to use.
*   `sdkconfig.defaults`: Default configuration options for ESP-IDF.
*   `wokwi.toml`: Configuration file for Wokwi ESP32 simulator.
